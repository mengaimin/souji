# 会社掃除当番システム

社員の当番順に基づき、5つの掃除（ゴミ捨て・トイレ・フロア・空気清浄機・給湯室）を自動ローテーションし、毎日 Slack に通知するシステムです。

## ローテーションのルール

1. **社員を登録**し、当番順（1番・2番・3番…）を決める
2. **ゴミ捨て**は順番どおり毎日交代（1番→2番→3番→…→全員終わったら1番へ）
3. **個人のタスク進行**: 今日ゴミ捨ての人は、明日トイレ → 明後日フロア → 空気清浄機 → 給湯室 と進む
4. 毎朝 Slack に「今日の5担当」「今日のゴミ捨て」「明日のゴミ回収」を通知

### 例（3人の場合）

| 日 | ゴミ捨て | トイレ | フロア | 空気清浄機 | 給湯室 |
|----|---------|--------|--------|-----------|--------|
| 1日目 | 1番 | 3番 | 2番 | 1番 | 3番 |
| 2日目 | 2番 | 1番 | 3番 | 2番 | 1番 |
| 3日目 | 3番 | 2番 | 1番 | 3番 | 2番 |

## セットアップ

### 1. Supabase プロジェクト

新規プロジェクトを作成するか、既存プロジェクトを利用します。

### 2. SQL 実行

Supabase SQL Editor で順に実行:

1. `souji.sql` … テーブル・ローテーション関数・RPC
2. `supabase_slack_cron.sql` … 毎日自動 Slack 送信（pg_cron）

**Extensions を有効化:**
- `pgcrypto`
- `http`
- `pg_cron`

### 3. index.html の設定

`index.html` 内の以下を自分の Supabase 情報に書き換え:

```javascript
const SUPABASE_URL = 'https://YOUR_PROJECT.supabase.co';
const SUPABASE_ANON_KEY = 'YOUR_ANON_KEY';
```

### 4. ブラウザで開く

`index.html` をブラウザで開くか、静的ホスティングにデプロイします。

### 5. 初期ログイン

- パスワード: `Admin1234`
- ログイン後、パスワード変更を推奨

### 6. Slack Webhook

1. Slack App → Incoming Webhooks を有効化
2. Webhook URL をコピー
3. 管理画面「設定」→ Slack 通知に貼り付け
4. 「テスト送信」で確認

## ファイル構成

| ファイル | 内容 |
|---------|------|
| `index.html` | Web アプリ本体 |
| `souji.sql` | DB スキーマ・ローテーション・Slack 送信 |
| `supabase_slack_cron.sql` | 毎日自動通知（pg_cron） |

## 運用のヒント

- **ローテーション起点日**: 運用開始日に合わせる（その日に1番がゴミ捨て）
- **社員の入退社**: 「社員・順番」で追加・削除・順番変更
- **通知時刻**: デフォルト JST 9:00（設定で変更可）
